# AI 内容排版网站静态框架设计

日期：2026-07-18
状态：待用户复核

## 1. 目标

制作一个仅面向个人使用的桌面端网站框架，用于展示“粘贴 AI 文本、调整排版、预览文档、导出 PDF”这一未来工作流的界面形态。

本阶段只实现静态视觉框架。所有按钮、输入区、选项、模板、缩放与导出入口均不实现实际功能。

## 2. 明确不做

- 不接入 AI API。
- 不生成或下载 PDF。
- 不实现文本编辑、自动排版、模板切换、缩放、保存或导出。
- 不实现账号、历史记录、云端存储、权限、计费或用量限制。
- 不实现后端服务、数据库或任务队列。
- 不做移动端与平板端适配。
- 不做性能或压力测试。
- 不为未确定的未来功能提前引入复杂抽象。

## 3. 页面结构

采用已确认的 A 方案“并排工作台”，页面限定为桌面端使用。

### 3.1 顶部操作栏

- 左侧显示产品标识“排版台”和当前文档名“未命名文档”。
- 右侧展示撤销、重做、保存草稿和导出 PDF 等静态操作入口。
- 所有入口只呈现视觉状态，不响应点击。

### 3.2 左侧内容区

- 展示“原始内容”标题与静态字数。
- 展示标题、粗体、斜体、列表和引用等静态格式工具。
- 使用示例中文内容表现正文编辑区域的排版效果。
- 底部展示静态“AI 排版建议”区域和“开始排版”按钮。

### 3.3 中间预览区

- 以浅灰工作台为背景，居中展示一张 A4 比例的白色纸张。
- 纸张内使用完整示例文章，展示标题、副标题、正文、引用和页码层级。
- 顶部展示页面、大纲和缩放等静态控件。
- 预览只用于表现视觉，不与左侧内容联动。

### 3.4 右侧设置区

- 展示版式、文字和页面三个静态标签。
- 展示模板缩略图、字体、字号、行距、内容密度和强调色等静态设置。
- 不实现选择、切换或实时更新。

### 3.5 底部状态栏

- 展示页面状态、A4 尺寸、页数和自动保存提示。
- 所有状态为固定示例文本。

## 4. 视觉系统

- 背景：雾灰色工作台，用于区分应用界面与白色纸张。
- 主文字：石墨色，保证长时间阅读的稳定性。
- 主操作色：青绿色，用于导出入口、选中状态与少量状态提示。
- 辅助强调色：珊瑚色，仅用于 AI 排版建议和文档引用。
- 纸张：纯白色，保持打印文档的真实感。
- 容器圆角控制在 8px 以内；不使用大面积渐变或装饰性图形。
- 控件使用紧凑的桌面工具界面尺度，避免营销页式的大标题和卡片堆叠。

## 5. 技术实现

- 使用 React + Vite 构建单页静态界面。
- 将应用外壳、顶部栏、内容区、预览区、设置区和状态栏拆分为小型组件。
- 使用本地静态示例数据，不创建状态管理层、请求层或服务端接口。
- 使用 CSS 变量维护颜色、边框、间距与字体等基础视觉值。
- 只保证常用桌面视口下的布局，例如 1280×800、1440×900 和 1600×900。
- 当浏览器宽度低于桌面设计最低宽度时，允许出现横向滚动，不提供响应式重排。

## 6. 验收标准

- 页面在本地开发环境可以正常打开。
- 桌面端三栏结构、顶部栏和底部状态栏与已确认视觉稿一致。
- 示例文档的标题、正文、引用与页码层级清晰。
- 配色、字体、边框、间距和控件密度形成统一视觉系统。
- 1280×800、1440×900 和 1600×900 视口下无主要内容重叠或意外截断。
- 项目可以完成生产构建。
- 不包含任何真实业务功能、移动端适配、后端代码或压力测试。

## 7. 后续原则

未来只有在明确提出某项功能时，才单独设计并实现该功能。当前静态框架不预设完整产品架构，也不为尚未确认的需求增加依赖或复杂度。
